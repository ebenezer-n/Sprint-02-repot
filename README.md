## 🎯 Objectif du projet

Créer un **CV en HTML5 sémantique pur**, sans aucune ligne de CSS ni de JavaScript.  
L'objectif est de démontrer la maîtrise de la structure et de la sémantique HTML5 : chaque balise doit avoir un sens, pas juste un rôle visuel.

---

```

```

---

## 🏗️ Structure HTML utilisée

| Zone HTML      | Rôle dans le CV                                      |
|----------------|------------------------------------------------------|
| `<header>`     | Nom, titre, photo de profil, contacts                |
| `<main>`       | Contenu principal du CV                              |
| `<aside>`      | Compétences, études, langues, hobbies (colonne gauche)|
| `<section>`    | Expériences, formations, diplômes (colonne droite)   |
| `<footer>`     | Liens vers les réseaux professionnels                |
| `<nav>`        | Navigation vers LinkedIn, GitHub, Email              |
| `<figure>`     | Photo de profil avec légende                         |
| `<article>`    | Chaque expérience ou diplôme individuel              |
| `<hr>`         | Séparation logique entre les grandes zones           |

---

## 🔖 Balises sémantiques clés

- `<strong>` — mise en valeur importante (noms d'entreprises, titres)
- `<em>` — emphase (dates, périodes)
- `<ul>` / `<li>` — listes non ordonnées (compétences, langues, hobbies)
- `<a href>` — liens externes (LinkedIn, GitHub) et mailto
- `<img alt>` — image avec texte alternatif pour l'accessibilité
- `<figcaption>` — légende de la photo de profil

## ♿ Accessibilité

- Attribut `alt` renseigné sur toutes les images
- Attribut `aria-label` sur la balise `<nav>` du footer
- Structure de titres hiérarchique (`h1` → `h2` → `h3`)
- Liens clairs et explicites (pas de "cliquez ici")
